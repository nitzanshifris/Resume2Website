# 🎯 Prompt לארגון מחדש של תיקיית Aceternity

## 📋 תיאור המשימה

אתה מקבל תיקיית פרויקט של Aceternity UI עם מבנה מסורבל וכפילויות רבות. המטרה שלך היא לארגן מחדש את התיקייה כך שתהיה ברורה, נקייה וקלה לשימוש באפליקציות אחרות.

## 🔍 מצב נוכחי - בעיות לפתרון

### 1. כפילויות במיקומים שונים:
- `/all_acenternity/` - רכיבים גולמיים
- `/component-library/` - מבנה מאורגן חלקית
- `/components/ui/` - רכיבי UI
- `/packs/` - ניסיון העברה לא שלם

### 2. בעיות מבניות:
- אותו רכיב מופיע ב-3-4 מקומות שונים
- חוסר עקביות בשמות (kebab-case vs PascalCase)
- ערבוב בין רכיבי בסיס לדמואים
- חוסר הפרדה ברורה בין סוגי רכיבים

## 📝 הנחיות לביצוע

### שלב 1: מיפוי וניתוח
1. סרוק את כל התיקייה וזהה:
   - כל הרכיבים הקיימים
   - כפילויות
   - תלויות בין רכיבים
   - רכיבים שבורים/לא שלמים

2. צור רשימה מקיפה של:
   - רכיבים ייחודיים
   - גרסה העדכנית ביותר של כל רכיב
   - תלויות נדרשות

### שלב 2: יצירת מבנה חדש
1. **צור את מבנה התיקיות החדש** - אבל אל תמחק את הישן!
2. **העתק רכיבים** - לא להעביר, רק להעתיק
3. **וודא שכל רכיב עובד** במיקום החדש

### שלב 3: ארגון רכיבים
1. **רכיבי Core** (`/src/components/core/`):
   - Button, Input, Card, Badge
   - רכיבים בסיסיים ללא תלויות מורכבות

2. **רכיבי UI** (`/src/components/ui/`):
   - Timeline, AnimatedTooltip, 3DCard
   - רכיבים מורכבים עם אנימציות

3. **Effects** (`/src/components/effects/`):
   - BackgroundBeams, Meteors, SpotLight
   - אפקטים ויזואליים

4. **Sections** (`/src/sections/`):
   - HeroSection, PricingTable, Testimonials
   - מקטעים מוכנים לשימוש

### שלב 4: יצירת Registry
```typescript
// registry/components.json
{
  "components": [
    {
      "name": "button",
      "type": "core",
      "path": "@/components/core/button",
      "dependencies": [],
      "description": "Basic button component",
      "tags": ["form", "interactive"]
    },
    // ... more components
  ]
}
```

### שלב 5: תיקון imports
1. עדכן את כל הimports לנתיבים החדשים
2. השתמש בpath aliases (@/components במקום ../../../)
3. וודא שהכל עובד

### שלב 6: תיעוד
1. צור README.md ראשי עם:
   - מבנה התיקיות
   - איך להשתמש ברכיבים
   - דוגמאות

2. לכל רכיב צור:
   - README.md עם הסבר
   - דוגמת שימוש
   - רשימת props

## 🔧 כלים מומלצים

### לסריקה ומיפוי:
```bash
# מצא כפילויות
find . -name "*.tsx" -o -name "*.ts" | xargs md5sum | sort | uniq -d -w 32

# מפה את המבנה
tree -I 'node_modules|.git|.next' > structure.txt

# חפש רכיבים
grep -r "export.*function\|export.*const" --include="*.tsx" --include="*.ts"
```

### לבדיקת תלויות:
```bash

grep -r "import.*from" --include="*.tsx" --include="*.ts" | sort | uniq
```
